Docker自动化部署以及测试
1. 通过交换模式进入容器中，启动服务，自动化测试，执行特定的脚本
2. 通过非交换模式执行容器中复制的文件，测试功能

> docker exec -it [container_id] bash 以交换式模式进入容器
> docker exec -it [container_id] sh   反之进入自定义的WORKDIR目录
  -it interactive terminal
  /app dir by default
> exit

> docker exec [container_id] sh -c './hello.sh' 执行容器中特定的脚本或者应用
